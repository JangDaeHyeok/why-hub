# 구현 스펙 — 평가 셋 / 골든 질의 (M5)

> **목적:** 유사 RAG 검색 품질을 **골든 질의 셋**으로 계량화해 recall/precision/MRR 을 추적한다.
> 기획안1 §14 로드맵의 M5. "평가 셋은 늦어도 M2 직후 작게라도" — 지금 코어(P01~P12) 위에 얹는다.
> **불변식·스코프:** 검색은 유사 RAG(벡터 없음) 그대로. 평가는 store/service 를 **읽기만** 한다.

---

## 1. 산출물 (모듈 배치)

```
hub/eval/
  golden.py     # 골든 코퍼스(문서) + 골든 질의(query→relevant ids)
  metrics.py    # recall@k / precision@k / reciprocal_rank / 집계 (순수 함수)
  harness.py    # evaluate(service, golden, k) → per-query + aggregate
scripts/eval_search.py   # CLI: 임시 store 에 코퍼스 시드 → 평가 → 리포트 출력
```

## 2. 골든 셋 계약

- **코퍼스:** lint 를 통과하는 마크다운 문서 목록. 키워드가 문서별로 구분되게 통제한다.
- **질의:** `{query, relevant: [doc_id...], filters?}`. `relevant` 는 그 질의에 대해 **맞다고 합의된**
  문서 id 집합(문서 단위 판정).
- 평가는 **격리된 임시 store** 에 코퍼스를 **save 루틴 경유**로 시드한 뒤 수행한다(파일 직접 쓰기 금지).

## 3. 지표 (문서 단위, 순위 = search 결과의 doc_id 를 중복 제거한 순서)

- **recall@k** = |top-k ∩ relevant| / |relevant|
- **precision@k** = |top-k ∩ relevant| / |top-k|  (top-k 는 실제 반환 수로 캡)
- **reciprocal_rank** = 첫 relevant 의 1/순위 (없으면 0), MRR = 평균
- **집계:** 질의 전체에 대한 각 지표의 평균 + 질의 수 n.

## 4. 수용 기준

- [ ] 지표 함수 단위 테스트(손계산 예시와 일치).
- [ ] 골든 집계: **recall@5 ≥ 0.9**, **MRR ≥ 0.8** (통제된 코퍼스에서 코어가 정상 동작함을 보증).
- [ ] **필터 선행 확인:** 필터로 배제되는 질의는 후보에서 빠진다(네거티브).
- [ ] **한국어 토크나이저 한계 문서화:** `unicode61` 이 조사를 분리 못 해 조사 결합 질의(예: "앵커를")가
      under-match 됨을 **테스트로 고정**한다([Δ] §7 후속 과제 근거).

## 5. 비범위

- 임베딩/하이브리드 검색(트리거 시 기획안2 G5). 커스텀 토크나이저(후속 과제).
- 대규모 코퍼스 벤치마크·CI 게이팅(초기엔 로컬 리포트로 충분).
