## 3. 실행 순서 (요청하는 법)

### 0단계 — 준비

- 위 §2대로 파일을 리포에 배치한다. **`CLAUDE.md`는 반드시 루트.**

### 1단계 — MVP 구현 착수 (지금)

1. `프롬프트-구현요청-팀내부MVP.md` **전체를 Claude Code에 프롬프트로 붙여넣는다.**
2. Claude Code가: 참조 문서 4개(CLAUDE.md·기획안1·delta스펙·Phase스펙)를 읽고 → **이해한 계획을 요약**
해 확인 요청 → 승인하면 **P01 구현 → 테스트 검증 → 요약 보고 후 정지.**
3. **한 Phase씩 진행한다.** 검토 후 "다음(P02) 진행"을 승인 → 반복. (P01 → P12)
4. **P09에서 워킹 스켈레톤 완성**(생성 → save → 검색 → 조회가 라이브러리·MCP·HTTP 전 경로로 도는 것).
5. 이어서 **M5**(평가 셋/골든 질의) → **M6·M7·M8**(UI: 읽기 → 쓰기 → AI 생성).
M8에서 `구현스펙-generate-M8.md`가 발동(프롬프트 문구는 골든 셋으로 튜닝).

> 요청은 사실상: **① MVP 프롬프트 1회 붙여넣기 → ② 완료된 Phase를 검토하고 "다음" 승인(반복).**
Phase마다 새 프롬프트를 붙일 필요는 없다(프롬프트가 Phase 루프를 이미 지시).
> 

### 2단계 — 범용 확장 (나중, 조건부)

- **전제:** MVP가 동작하고, 승격 트리거(팀 밖 사용 / 세밀 권한 필요 / FTS 후보 품질 저하 / 다중 소비자)
중 하나가 **실제로** 발생.
- 방법: `프롬프트-구현요청-범용확장.md`를 붙여넣는다.
- Claude Code가 G1부터: **마일스톤별 상세 스펙·Phase 분해를 먼저 작성**(MVP에서 했던 것처럼 스펙 선구동)
→ 승인 후 구현. (G1 권한 → G2 커넥터 → G3 배포/관리 UI → G4 쓰기 도구 → G5 조건부 벡터)

## 5. 확정 사항 요약 (빠른 참고)

- 저장 백엔드: **pluggable**(File=SQLite FTS5 · Postgres=tsvector, 배포) · 앵커: **git diff 방식**(줄 diff → 최근접 헤더 귀속 → slug 유일화)
- save 동시성: File=**파일 락** / Postgres=**advisory lock + 트랜잭션** · 이력: **자체 스냅샷 + delta**(git 없음)
- 검색: **유사 RAG**(벡터/GPU 없음) · 쓰기: **3경로(세션 포크 · UI-AI · UI 직접) → 단일 save 루틴**
- UI: **커스텀 경량(FastAPI + HTMX)** + 편집/AI 생성 뷰 마크다운 에디터 · LLM: **OpenAI 호환, 호스팅 API 우선**
- 벡터/멀티테넌트/권한 강제 = **기획안 2**(트리거 발생 시). FTS 스키마엔 `project/tenant` 컬럼 미리 개설.

## 6. 배포 (docker-compose)

저장 백엔드는 pluggable(`hub/store/base.py`): **file**(기본·로컬/테스트: 파일+SQLite FTS5) / **postgres**(배포).
배포는 모든 상태를 PostgreSQL 에 두고 MCP 는 HTTP(streamable-http)로 노출한다.

```
# 시크릿·엔드포인트는 .env 로 (PGPASSWORD, KNOWLEDGE_HUB_LLM_COMPLETE_URL/_STREAM_URL) — .env.example 참고
cp .env.example .env
docker compose up --build              # postgres + admin(:8000) + mcp(:8001)
docker compose run --rm admin python scripts/import_to_postgres.py   # 기존 knowledge/ 1회 이관
```

- 관리 서버: `http://localhost:8000`(웹 UI·HTTP API·승인함). MCP: `http://localhost:8001`(streamable-http).
- 배포 설정: `config.deploy.toml`(`[storage] backend="postgres"`). 로컬은 `config.toml`(file) 유지.
- 상세: `docs/specs/구현스펙-postgres-배포.md`.
