# 워크플로우 — 세션 포크로 ADR 캡처하기 (도그푸딩)

> **목적:** Claude Code에서 설계 대화를 나눈 뒤, **메인 세션의 컨텍스트를 버리지 않고** 그 결정을
> ADR/설계의도 문서로 남긴다. 이 문서만 보고 제3자가 그대로 따라 할 수 있게 자기 완결적으로 쓴다.
> 참조: 기획안1 §6, CLAUDE.md §9.

---

## 왜 "세션 포크"인가

- 세션 포크는 **대화 이력만 분기**하고 **파일시스템은 분기하지 않는다.** 포크가 파일을 쓰면 그 파일은
  실제로 남고 원본 작업 디렉토리에서도 보인다.
- 우리에겐 이 성질이 정확히 유리하다 — **문서(파일)는 남기고, 초안 작성으로 오염된 대화 세션만 버리면**
  메인 세션의 컨텍스트가 보존된다. 포크의 이점은 *파일 격리가 아니라 대화 컨텍스트 보존*이다.

## 절차 (한 번에 하나)

1. **설계 대화를 충분히 나눈다** (메인 세션). 결정·대안·트레이드오프가 대화에 드러날 때까지.
2. **포크한다.** 세션 안에서 `/branch`, 또는 CLI에서
   `claude --resume <id> --fork-session` / `claude --continue --fork-session`.
   원본 세션은 그대로 남고 새 세션 ID가 부여된다.
   (버리는 초안이면 print 모드의 `--no-session-persistence` 도 가능.)
3. **포크에게 초안을 지시한다.** "이 맥락으로 `templates/adr.md`(또는 `design-intent.md`) 골격을 채운
   초안을 써라. **배경·결정·근거·대안·결과**를 모두 채우고, **대안에는 폐기한 선택지와 이유**를 반드시 넣어라."
4. **사람이 검토·수정한다.** Why(특히 대안·폐기 선택지)가 실제로 채워졌는지 확인한다.
5. **저장한다 — 반드시 save 루틴 경유.** 다음 중 하나로 `save_document` 를 호출한다(직접 파일 쓰기 금지):
   - MCP 도구 `save_document(markdown, actor, [intended_diff])`
   - HTTP `PUT /docs/{id}` (body: `{ "markdown": ..., "actor": ... }`)
   save 루틴이 정규화 → **lint 게이트** → 스냅샷 diff → 이력 append → 재색인을 자동 처리한다.
   **lint 실패(예: 대안 비어 있음)면 저장이 거부되고 사유가 반환된다.** 사유를 고쳐 다시 저장한다.
6. **포크 세션을 폐기한다.** 문서(파일)는 남고 대화 분기만 버려져 **메인 세션 컨텍스트가 보존**된다.

## 스펙 선구동 옵션 (docs-diff)

큰 변경은 먼저 **의도된 변경**을 남길 수 있다. `save_document` 의 `intended_diff` 인자로 넘기면
`docs-diff/<id>.<date>.md` 에 기록되고 `GET /docs/{id}/diff` 로 조회된다. 나중에 자동 delta(실제 변경)와
대조해 **의도 대비 드리프트**를 잡는다.

## 체크리스트

- [ ] 대안 섹션에 **폐기한 선택지 + 이유**가 있는가 (없으면 저장 거부).
- [ ] 결정이 한 문장으로 명확한가.
- [ ] `save_document` 경유로 저장했는가 (파일 직접 수정 아님).
- [ ] 포크 세션을 폐기해 메인 컨텍스트를 보존했는가.
